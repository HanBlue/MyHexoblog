从源码理解SpringBoot自动配置原理

SpringBoot 源码分析
学习笔记
发布日期:   2021-10-08

一、简单理解 SpringBoot

Spring 组件一站式解决方案，主要是简化了使用 Spring 的难度，简省了繁重的配置，提供了各种启动器，开发者能快速上手。

1、独立运行

内嵌了各种servlet容器，Tomcat、Jetty等，现在不再需要打成war包部署到容器中，Spring Boot 只要打成一个可执行的jar包就能独立运行，所有的依赖包都在一个jar包内。

2、简化依赖

Starters包含了许多项目中需要用到的依赖，可以集成 Spring 及其他技术，它们能快速持续的运行。

3、无代码生成和XML配置

Spring Boot 配置过程中无代码生成，也无需XML配置文件就能完成所有配置工作。

4、应用监控

Spring Boot 提供一系列端点可以监控服务及应用，做健康检测。

二、SpringBoot 的核心文件和核心注解

Spring Boot 的核心配置文件是 application 和 bootstrap 配置文件。前者主要用于 Spring Boot 项目的自动化配置，后者主要用于Spring Cloud Config 配置中心加载外部配置与一些加密解密的场景。

Spring Boot 的核心注解是@SpringBootApplication，主要组合包含了以下 3 个注解：

@SpringBootConfiguration：实现配置文件的功能。

@EnableAutoConfiguration：打开自动配置的功能。

@ComponentScan：组件扫描。

三、SpringBoot 自动配置原理

总结放前面：在main方法中添加 @EnableAutoConfiguration 开启自动配置功能，该注解中自动去maven中读取每个starter中的spring.factories文件映射的自动配置的类。

自动配置的类的核心是@Configuration和@ConditionalOnClass，声明了配置文件并根据类路径下是否有这个类去自动配置。

最终发现是加载的META-INF/spring.factories文件，找到project下的External Libraries中的org.springframework.boot.autoconfigure，然后打开，可以看到一系列Spring Boot自动配置的列表。

最后我们看到了自动配置redis的类

1）@ConditionOnClass 激活一个配置，当类路径中存在这个类时才会配置该类
2）@EnableConfigurationProperties 自动映射一个POJO到Spring Boot配置文件（默认是application.properties文件）的属性集。
3）@ConditionalOnMissingBean 启用一个Bean定义，但必须是这个Bean之前未定义过才有效。

最终我们了解了，添加 @EnableAutoConfiguration 后会开启自动配置功能，去maven中读取每个starter中的spring.factories文件映射的自动配置的类。自动配置的类的核心是@Configuration和@ConditionalOnClass，声明了配置文件并根据类路径下是否有这个类去自动配置。
