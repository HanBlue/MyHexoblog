1． ElasticStack

1.1简介

ELK是一个免费开源的日志分析架构技术栈总称，官网 https://www.elastic.co/cn

包含三大基础组件，分别是Elasticsearch、Logstash、Kibana。但实际上ELK不仅仅适用于日志分析，它还可以支持其它任何数据搜索、分析和收集的场景。

随着ELK的发展，又有新成员 Beats、elastic cloud 的加入，所以就形成了 ElasticStack。架构图如下

1.2 特点

  • 处理方式灵活：elasticsearch 是目前最流行的准实时全文检索引擎，具有高速检索大数据的能力。
  • 配置简单：安装elk的每个组件，仅需配置每个组件的一个配置文件即可。修改处不多，因为大量参数已经默认配在系统中，修改想要修改的选项即可。
  • 接口简单：采用 json 形式RESTFUL API接受数据并响应，无关语言。
  • 性能高效：elasticsearch 基于优秀的全文搜索技术Lucene，采用倒排索引，可以轻易地在百亿级别数据量下，搜索出想要的内容，并且是秒级响应。
  • 灵活扩展：elasticsearch 和 logstash 都可以根据集群规模线性拓展，elasticsearch 内部自动实现集群协作。
  • 数据展现华丽：kibana 作为前端展现工具，图表华丽，配置简单。

1.3 组件

Elasticsearch

基于 java 开发，基于Lucene、分布式、通过 Restful 方式进行交互的近实时搜索平台框架。

Logstash

基于 java 开发，是一个数据抽取转化工具。

Kibana

基于 node.js 开发，也是一个开源和免费的可视化工具。

Beats

集合了多种单一用途数据采集器。

2. Elasticsearch

2.1 简介

全文检索、倒排索引和 Lucene

倒排索引是指数据存储时，经行分词建立 term 索引库，这种索引表中的每一项都包括一个属性值和具有该属性值的各记录的地址。

全文检索就是使用倒排索引进行查找，而 Lucene 就是一个jar包，里面封装了全文检索的引擎、搜索的算法代码。

Elasticsearch 就是基于 Lucene 的搜索服务器，基于 REST 风接口提供了一个分布式多功能搜索引擎，也是是当下最流行的企业级跨平台开源搜索引擎。

Elasticsearch 能够进行全文检索、结构化检索、数据分析、分布式、近实时对巨量数据进行处理，适用于互联网搜索、网站站内搜索、站内数据分析等多个场景。

简称为 es，可以看做一个分布式 nosql 数据库，内部采用版本号机制（乐观锁），在竞争不强的情况下性能非常好。

2.2 核心概念

Near Realtime：近实时

写入数据时，过1秒才会被搜索到，因为内部在分词、录入索引。es 搜索时：搜索和分析数据需要秒级出结果。

Cluster：集群

包含一个或多个启动着 es 实例的机器群。通常一台机器起一个 es 实例。同一网络下，集名一样的多个es实例自动组成集群，自动均衡分片等行为。默认集群名为“elasticsearch”。

Node：节点

每个 es 实例称为一个节点。节点名自动分配，也可以手动配置。

Index：索引

对应关系型数据库中的数据库和表。

索引创建规则：

  • 仅限小写字母
  • 不能包含\、/、 *、?、”、<、>、|、#以及空格符等特殊符号
  • 从7.0版本开始不再包含冒号
  • 不能以-、_或+开头
  • 不能超过255个字节（注意它是字节，因此多字节字符将计入255个限制）
Document：文档

对应数据库中的一条记录。通常以json格式显示。多个document存储于一个索引（Index）中。

Field：字段

对应数据库中的列，定义每个document应该有的字段。

Type：类型

默认 _doc ，官方在淡化概念。

shard：分片

index 数据过大时，将index里面的数据，分为多个shard，分布式的存储在各个服务器上面。可以支持海量数据和高并发，提升性能和吞吐量，充分利用多台机器的cpu。分片自动负载均衡，分片向空闲机器转移。

replica：副本

分布式为保证可用性，将每个index的分片经行备份，存储在另外的机器上。能正常提供查询和插入的分片我们叫做主分片（primary shard），其余的叫做备份的分片（replica shard）。主分片的数量无法改变，但是副本的数量并不固定，同时，副本能够提供查询功能，这样也变相减少了服务器压力。

es6默认新建索引时，5分片，2副本，也就是一主一备，共10个分片。所以，es集群最小规模为两台。

路由算法
shard = hash(routing) % number_of_primary_shards

哈希算法的一种，对主分片数进行取模来决定数据分到哪个分片。

也可以自己指定 routing number 控制数据流向，好处是比较灵活，但如果设计的不好会导致数据倾斜。

PUT /test_index/_doc/15?routing=num
{
  "num": 0,
  "tags": []
}
mapping：映射

自动或手动为index中的_doc建立的一种数据结构和相关配置，mapping中包含了每个field对应的数据类型，以及如何分词等设置。

搜索行为

不同的 data type 的分词、搜索等行为是不一样的。有 exact value 精确匹配与 full text 全文检索两种，前者要求完全一样才能显示，后者对值进行拆分词语后（分词）进行匹配，也可以通过缩写、时态、大小写、同义词等进行匹配。

analyzer：分词器

切分词语，normalization（提升recall召回率），将一段文本进行各种处理，最后处理好的结果才会拿去建立倒排索引。有很多类型，可以使用默认的，也可以指定。

核心数据类型

string 转成 text or keyword ，full text

byte，short，integer，long,float，double，boolean 转成 date，exact value

2.3 安装

Windows 安装 elasticsearch

1、安装JDK，至少1.8.0_73以上版本，验证：java -version。

2、下载和解压缩Elasticsearch安装包，安装后配置文件在 config 目录下的 elasticsearch.yml、jvm.options 和log4j2.properties 内。

3、启动Elasticsearch：bin\elasticsearch.bat，es的特点就是开箱即，无需配置，启动即可。

4、检查ES是否启动成功：浏览器访问 http://localhost:9200/?pretty

5、浏览器访问 http://localhost:9200/_cluster/health 查询集群状态

Windows 安装 Kibana

1、kibana 是 es 数据的前端展现，数据分析时，可以方便地看到数据。

2、下载，解压 kibana。

3、启动 Kibana：bin\kibana.bat

4、浏览器访问 http://localhost:5601 进入DevTools 界面，支持代码提示。

2.4 操作命令

1、集群管理

// 快速检查集群的健康状况

GET /_cat/health?v

green：每个索引都是active状态的。

yellow：部分replica shard不是active状态。

red：不是所有索引的primary shard都是active状态的，部分索引有数据丢失了。

//快速查看集群中有哪些索引

GET /_cat/indices?v

2、文档 CRUD 操作

//新建索引

语法：PUT /index

PUT /book

//新增文档

语法：PUT /index/type/id

新增图书：PUT /book/_doc/1{“name”: “Bootstrap开发”,”price”:38.6,}

//检索文档

语法：GET /index/type/id

查看图书：GET /book/_doc/1 就可看到 json 形式的文档。

//检索返回特定字段

返回图书价格：GET /book/_doc/1?__source_includes=name,price

//修改文档

语法：POST /{index}/_update/{id}

修改图书的名称：POST /book/_update/1/ { “doc”: { “name”: “ Bootstrap开发教程高级” }}

//覆写文档（一般不用，有性能以及并发冲突问题）

PUT /book/_doc/1

//删除文档

DELETE /book/_doc/1

3、内置脚本

es可以内置脚本执行复杂操作，例如查询同时更新。

//修改文档6的 num 字段+1

POST /test_index/_doc/6/_update
{
“script” : “ctx._source.num+=1”
}

//搜索文档并将 num 字段*2
GET /test_index/_search

{
  "script_fields": {
    "my_doubled_field": {
      "script": {
       "lang": "expression",
        "source": "doc['num'] * multiplier",
        "params": {
          "multiplier": 2
                }
            }
        }
    }
}

4、手动管理mapping

//查询索引的映射

语法：PUT book/_mapping

//创建映射
PUT book/_mapping
{
    "properties": {
           "name": {
                  "type": "text"
            },
           "pic":{
             "type":"text",
             "index":false
           }
    }
}

通过analyzer属性指定分词器。keyword关键字字段不进行分词，日期类型也不用设置分词器，数值类型尽量选择范围小的类型，提高搜索效率。

index属性指定是否索引，像是图片的 index 就设置为 false。

//删除映射

删除索引就行。

//修改映射

无法修改，只能通过删除索引重新创建映射来进行操作。

5、复杂数据类型

//多重字段

{ “tags”: [ “tag1”, “tag2” ]}

//空字段

null，[]，[null]

//对象字段
PUT /company/_doc/1

  "address": {
    "country": "china",
    "province": "guangdong",
    "city": "guangzhou"
  },
  "name": "jack",
  "age": 27,
  "join_date": "2019-01-01"
}
