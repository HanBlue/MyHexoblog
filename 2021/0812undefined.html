毕业设计部署到服务器

毕业设计 成果展示
项目记录
发布日期:   2021-08-12

1，为什么要部署到云服务器

没有上线的项目只能在本地访问，想要让其他人访问就必须购买远程服务器，然后将项目上传并完成配置，这样就可以通过你购买的服务器的IP进行访问。

如果还想要其他人直接通过网址进行访问，就需要申请管局备案，流程根据购买服务的供应商不同而有所差别，我是通过腾讯云渠道提交备案材料的，需要准备身份证并且填写网站资料，随后还有客服电话沟通修正资料等等。

2，准备工具

1，已经调试并且打包好的本地项目

这个不是压缩，是通过工具将前端转化成 dist 静态文件夹，后端转化为 jar 包。我因为后端使用的是Spring Cloud，需要打包的工程也比较多。

2，Xshell 软件

在 Windows 系统里远程控制 Linux 所使用的软件，后面用来启动服务器里的服务。

3，宝塔面板

简单好用的 Linux 服务器运维管理面板，方便安装程序、创建数据库、文件上传、配置反向代理等等。

3，部署流程

1，购买云服务器

腾讯云有学生认证优惠，平常时候比较值得，如果是双11或者有活动也可以考虑其他的。

我购买的是1核2G的，部署一个项目以及少量访问刚刚好，半年54；域名买的top类型，几块钱一年。

2，连接服务器

购买云服务器之后，进入控制台页面，复制IP地址，设置SSH密钥，在 Xshell 中登录。

Xshell 可以远程控制Linux的服务器，类似于控制台。

3、安装宝塔，配置环境

进入宝塔官网，看到 Centos 安装脚本，复制命令到 Xshell 中安装，安装好后看到外网登录地址和账号密码，在浏览器中打开登录。

服务器中需要配置各种环境，主要是方便以后的项目运行。

按照提示自动安装各种软件，比较重要的是 Nginx 和 MySQL，放到首页比较方便。

创建数据库，导入本地数据库的脚本。

配置 jdk 环境变量，这个跟 windows 里一样都是为了方便java程序运行。

放行要用的端口，不然无法访问端口服务。

配置Nginx反向代理实例，配置完作为实例后面备案也要用。

4，上传前后端打包的文件，联调

在宝塔面板中创建新网站，自动生成网站文件夹，在本地把前端项目用 webpack 打包成 dist 文件夹然后上传。把所有后端工程用 maven 打包成 jar 包然后上传。

启动后端服务，在 Xshell 中输入 java -jar jarName-0.0.1-SNAPSHOT.jar，由于spring cloud的微服务架构我启动了六个微服务，这里就不展示了。

启动服务后，你就可以通过 Nginx 中配置的端口进行访问了，查看访问有没有问题，有的话关闭服务、在本地修改、打包、再上传启动。

如果调试完成，就可以通过 nohup java -jar jarName-0.0.1-SNAPSHOT.jar >msg.log 2>&1 & 命令后台启动服务，后台启动的进程不会随着控制台的关闭结束。

5，备案

没有备案的网站只能通过网址+端口的形式访问，无法通过Nginx进行80端口转发，比较low。想要别人直接通过 www.hanblue.top 这种形式的，乖乖去备案吧，管局审核大概需要20天时间。

4，经验总结

第一次部署网站，配置环境使用各种工具很方便，难的是上传之后的前后端联调，如果在本地没有完全修改好涉及到网关、数据库、密钥的文件，那到远程服务器运行会令人更加头大如斗，以后一定要再三确认后部署。
